"Why separate training from inference?"
  â†’ Training is expensive (100ms+). We train once offline, serialize the model,

"Why use Pydantic validation?"
  â†’ Fail fast at boundaries. Invalid input rejected before reaching model.
    Prevents silent failures and provides auto-documentation.

"Why load model at startup (not per-request)?"
  â†’ Loading model takes 100ms. With 100 predictions/day, that's 10 seconds
    wasted. Single load at startup is 1-2 seconds total + 5ms per prediction.

"Why Random Forest?"
  â†’ Better accuracy (92% vs 89%) handles non-linearity automatically.
    2-3ms latency increase justified by 3% accuracy improvement.

"How would you scale this?"
  â†’ Phase 1: Multi-worker FastAPI (10x)
    Phase 2: Kubernetes (100x)

  â†’ Monitor prediction accuracy on recent holdout data. Retrain monthly.
